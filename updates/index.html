App release notes and changes.

Version 1 · Build 4

Race times, GPS fixes, and monochrome refresh

Ranked race times now show correctly across all screens. Match history shows elevation credit breakdowns. New info button explains MMR, tiers, and DNF rules. GPS false-start bug fixed. Run History and Friends screens refreshed to match the monochrome theme. Social feed polished. In-app friend request notifications added. Add-friend button now on profiles.

Released June 11, 2026

Run History

  • Refreshed the run history screen to better match Jeema's current monochrome theme.
  • Removed the large Run History title from the sheet.
  • Updated the Done button to match the compact style used across other profile sheets.
  • Added stronger black accents to each run card so distance, time, pace, and run type are easier to scan.
  • Kept the page background light so the screen feels consistent with the rest of the app.

Friends

  • Refreshed the Friends tab with a cleaner monochrome look.
  • Replaced the old dark green styling with light grey backgrounds, white cards, black buttons, and clearer row borders.
  • Search results, pending requests, and friend rows now use a more consistent card layout.
  • Friend avatars and action buttons now better match the current Jeema theme.

Friend Requests

  • Jeema can now show a notification when a new friend request is detected while the app is running.
  • Friend request notifications use the sender's username so it is clear who wants to connect.
  • Foreground notifications can now show as banners instead of being silently hidden while the app is open.

Ranked Match History

  • Match history now shows when elevation has added ranked distance credit.
  • Match details now include a clearer breakdown of GPS distance, elevation credit, and ranked effort distance.
  • This should make hilly ranked runs easier to understand when the GPS distance appears slightly under 5 km.
  • Fixed a race condition where match history could reload before the ranked result had finished writing to the server, causing the latest run to appear missing until the next app launch.

Ranked Data Reliability

  • Fixed a bug where wins and losses could revert to a stale count after a ranked run if the result failed to write to the server. Wins and losses now always keep the highest value seen, so a network hiccup can no longer undo a recorded result.
  • Fixed app launch sync to use the higher of two data sources when reconciling wins and losses, so a missed write on one path cannot overwrite a correct count from the other.
  • Ranked results that fail to upload due to poor signal are now saved locally and retried automatically the next time a connection is available. No ranked run result should be permanently lost due to a bad connection.

Ranked System Info

  • Added an info button to the top-right corner of the ranked card. Tapping it opens a sheet explaining what MMR is, how the 5 km ghost race format works, how MMR changes are calculated, and the full tier structure from Tortoise to Falcon.
  • The ranked info sheet now includes a "FINISHING & DNFs" section explaining that you must physically complete the full 5 km to finish a race, that stopping before 5 km is a DNF and counts as a loss, and that continuing to run after 5 km still tracks all stats.

Ranked Race Times

  • The bonus running screen now shows your locked 5 km race time on the result card while you continue running.
  • The end-run result screen now shows a RACE TIME / TOTAL RUN split when you ran past 5 km, so your finish time is never confused with your total run time.
  • The social feed now shows RACE TIME for ranked posts. If the runner continued past 5 km, both RACE and TOTAL times are shown.
  • Match history rows now display the locked 5 km finish time instead of the total run duration.
  • Match history detail view now shows RACE TIME (and TOTAL TIME when applicable). Pace is now calculated from the 5 km race time rather than the full run duration.

Social Feed

  • Like and comment buttons on the social feed now use white for better contrast against the dark reel background.
  • Liking a post now triggers haptic feedback. The same feedback fires when double-tapping to like.
  • Fixed an issue where some users' avatars were not showing on the social feed even though the user had a profile photo set. Avatar and username lookups are now handled by separate focused queries, which avoids a PostgREST behaviour where fetching both columns together could silently drop the avatar for some rows.

Member Duration

  • Fixed the "MEMBER FOR X DAYS" counter showing one day short. A user who joined today now correctly shows "MEMBER FOR 1 DAY".

Friend Profiles

  • An "ADD FRIEND" button now appears on a user's profile when you are not yet friends. Tapping it sends a friend request.
  • If a request is already pending, the button changes to "REQUEST SENT · TAP TO CANCEL" so you can withdraw it.
  • If the other person has already sent you a request, a banner makes that clear.

GPS Distance Accuracy

  • Fixed a bug where runs could show an extra 30–50 metres of false distance at the start. iOS delivers stale cached location fixes immediately when tracking begins, and these were being counted as real movement. The app now discards any fix whose timestamp predates the moment tracking started.
  • The same stale-fix protection now applies after unpausing a run, preventing a burst of false distance from a cached location being measured against the pre-pause position.
  • Tightened the GPS accuracy filter so only fixes with less than 25 metres of horizontal uncertainty are used for distance calculations (previously up to 50 metres was accepted). This reduces the path zigzag that caused Jeema to read ahead of other apps like Strava over longer runs.
Version 1 · Build 3

Ranked runs, leaderboards, profiles, and post-run polish

This update focuses on ranked runs, leaderboards, player profiles, and post-run polish.

Released June 5, 2026

Permissions

  • Location and notification permissions are now handled during first-launch setup.
  • Run screens no longer trigger surprise permission popups when starting a run.
  • If location permission is not fully enabled, the setup screen now lets users retry or open Settings.
  • If iOS does not show the Always Allow popup, Jeema now gives clearer instructions to enable it from Settings.
  • Notifications remain optional, but Always Allow location is required for reliable run tracking.

Ranked Runs

  • Players now appear on the leaderboard after completing their first placement run.
  • Your first placement run now counts toward your running stats.
  • Ranked runs now save to your stats as soon as the 5 km ranked result is completed.
  • If you continue running after the ranked 5 km, the extra distance still updates the saved run.
  • Apple Watch ranked completions now count toward stats even when only the ranked finish time is received.

Post-Run Results

  • The ranked post-5 km screen has been simplified.
  • Result cards now show a clear VICTORY or DEFEAT state instead of comparing your time against the opponent's time.
  • Win and loss cards now use softer theme-friendly green and red colors.
  • Defeat text and supporting text are easier to read.
  • The post-run copy now uses Jeema branding.
  • The share-to-feed ranked result screen no longer uses the old gold theme.

Leaderboards

  • Leaderboard visibility now starts after one placement run.
  • Global, regional, friends, and division leaderboards all use the updated placement requirement.
  • Tapping players from the leaderboard now opens their public profile view, not just friend profiles.

Player Profiles

  • Player profiles now include a fuller stats view.
  • Profile stats now show mileage, season/all-time stats, elevation, time run, streaks, and personal bests.
  • The mileage card on player profiles now matches the stats page design.
  • Public player profiles no longer show private or editable goal controls.

Previews And Development Fixes

  • Fixed ranked run previews that were failing because they depended on live location tracking.
  • Fixed run history previews that were crashing from missing preview environment data.
  • Added a preview for the ranked share-to-feed result screen.

Visual Cleanup

  • Removed more of the old gold styling from ranked result screens.
  • Updated ranked result colors to better match the current app theme.
